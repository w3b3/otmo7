— 06.2 / continued from 06

We flag, measure, and decide.

01

Decouple deploy from release. Code goes to production behind feature flags. Exposure is a separate, controlled decision.

02

Ship dark, then light up gradually. Internal first. Slice of users next. Watch the signals. Roll forward or roll back based on what you see.

03

Run experiments where they matter. When two reasonable approaches exist and the answer isn't obvious, set up the comparison and let real usage decide.

04

Decisions are data-backed by default. In architecture review, the first question on any proposed change is "what did the telemetry show?" — not "what do you think?"
